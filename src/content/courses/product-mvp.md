Great software starts before the first production feature. This course teaches how to get behind the customer's world: observe behavior, run switch-style interviews, map direct / indirect / legacy / analog competition, and write a clear problem statement.

You will stage validation in three moves — **problem**, **solution**, then **willingness to pay** — and produce rough prototypes on purpose (so feedback stays honest). Use AI to synthesize research notes, draft interview guides, and spin early Figma or code sketches faster; you still decide what is worth building.

Capstone: a portfolio artifact that shows **evidence**, not just features — problem statement, landscape map, and an MVP prototype a real user can react to.
